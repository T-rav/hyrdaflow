# discover-hid-ambiguity

The issue body is peppered with ambiguity markers ("maybe", "not sure",
"could be", "it depends") but the brief's *Open questions* section
literally says `(none)`. The Discover Completeness rubric (criterion 5)
forbids claiming zero open questions when the input contains ambiguity
markers — the brief must surface at least one explicit question.

Keyword: hid-ambiguity
